## 一、跨文档实体关系抽取

### 1.1 功能概述
实现了一个跨文档实体关系抽取功能页面，允许用户选择两个实体（公司），系统自动查找包含这两个实体的新闻并抽取它们之间的关系。

### 1.2 页面位置
- 文件：`front/src/views/CrossDocEntityExtract.vue`
- 路由：`/cross-doc-extract`
- 导航菜单：跨文档抽取 → 跨文档实体关系抽取

### 1.3 功能特性

#### 实体选择
- 支持远程搜索公司实体
- 双实体输入（实体1和实体2）
- 智能搜索提示

#### 三元组展示
- 单条三元组展示（实体1 - 关系 - 实体2）
- 左右箭头切换不同三元组
- 显示当前位置计数（如 1/3）
- 淡色透明导航箭头，支持循环浏览

#### 关系入库
- "加入图谱"按钮，将当前展示的三元组添加到知识图谱
- 支持逐条确认入库

#### 新闻展示
- 展示相关新闻标题、来源、时间
- 新闻摘要高亮显示
- 支持查看原文链接

### 1.4 案例数据
使用真实案例数据（来自 `lsydata/data/financedata/完整_已知实体匹配_关系预测结果1/案例.xlsx`）：
- **实体1**: 爱迪特（秦皇岛）科技股份有限公司
- **实体2**: 科美诊断技术股份有限公司
- **关系**: 起诉
- **新闻标题**: 深圳迈瑞状告成都迈瑞、徐州迈瑞
- **时间**: 2023-04-21 18:53:00
- **来源**: 腾讯网

### 1.5 后端API接口需求

| 接口 | 方法 | 说明 |
|------|------|------|
| `/search_companies/?keyword=xxx` | GET | 搜索公司实体 |
| `/cross_doc_extract/` | POST | 执行跨文档实体关系抽取 |
| `/add_triples_to_graph/` | POST | 将三元组添加到知识图谱 |

**请求/响应格式详见代码注释**

## 二、证据增强决策（已隐藏）

### 2.1 功能概述
实现了一个基于时序路径证据的事实判定功能。用户输入候选四元组（实体1、实体2、关系、目标时间），系统自动查找相关的时序路径证据，并根据证据判断该事实是否在目标时间成立。

### 2.2 页面位置
- 文件：`front/src/views/EvidenceEnhancedDecision.vue`
- 路由：`/evidence-decision`
- 导航菜单：证据决策 → 证据增强决策（当前已隐藏）

### 2.3 功能特性

#### 四元组输入
- 实体1（起始实体）
- 实体2（目标实体）
- 关系类型
- 目标时间（日期选择器）

#### 时序路径证据
- 系统自动查找相关时序路径
- 路径默认隐藏，点击"查看时序路径证据"按钮展开
- 可视化展示路径（实体 → 关系[时间] → 实体）
- 显示路径有效性标签

#### 判定结果
- 离散判定结果：成立/不成立
- 判定理由时间线
- 每条路径的详细分析

### 2.4 当前状态
**该功能已在前端导航中隐藏**，如需启用，请修改 `HomeView.vue` 取消注释相关菜单项。

### 2.5 后端API接口需求

| 接口 | 方法 | 说明 |
|------|------|------|
| `/evidence_decision/` | POST | 基于四元组进行事实判定 |

## 五、使用说明

### 5.1 启动前端开发服务器
```bash
cd front
npm run serve
```

### 5.2 访问功能
1. 登录系统
2. 在左侧导航栏选择"跨文档抽取"
3. 点击"跨文档实体关系抽取"
4. 选择两个实体，点击"开始抽取"
5. 查看抽取结果，可通过左右箭头切换三元组
6. 点击"加入图谱"将三元组添加到知识图谱

### 5.3 启用证据决策功能（如需）
编辑 `front/src/views/HomeView.vue`，取消以下代码块的注释：
```vue

</el-sub-menu> -->
```

同时取消组件导入和注册的注释。
