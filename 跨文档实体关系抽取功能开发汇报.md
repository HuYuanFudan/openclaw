# 跨文档实体关系抽取功能开发汇报

---

## 一、项目背景与目标

### 业务需求
从财经新闻中自动识别两家公司之间的关系，辅助金融分析师快速了解企业关联网络。

### 核心目标

| 目标 | 说明 |
|------|------|
| 实体选择 | 支持用户选择任意两家公司进行分析 |
| 关系抽取 | 从新闻文本中自动识别关系类型 |
| 结果展示 | 以可视化方式呈现三元组及证据 |
| 数据溯源 | 展示支撑关系的原始新闻内容 |

---

## 二、数据准备与处理

### 原始数据规模

- **新闻总量**：20,000 条财经新闻
- **涉及公司**：117 家主体公司
- **时间跨度**：2024-2025 年

### 数据质量问题发现

在处理过程中发现一个关键问题：原始数据中的"关联"标记存在大量误判。

| 数据状态 | 关系条数 | 说明 |
|----------|----------|------|
| 原始标记 | 3,044 条 | 仅表示两家公司在同一篇新闻中出现 |
| 真实关系 | 约 320 条 | 经过滤后确认有实际语义关联 |

**典型误判案例**：

- 《重大事项公告汇总》类新闻，将十几家无关联公司并列在一起
- 《上市公司股价波动快报》类新闻，按涨跌幅排名，公司间并无关系

### 解决方案：智能过滤算法

设计了**三重过滤机制**：

```
原始新闻 (20,000条)
    ↓
第一层：同句共现检测
    ↓ 过滤掉 88% 的假关系
候选关系 (约 1,600 条)
    ↓
第二层：关系关键词匹配
    ↓ 标注具体关系类型
确定关系 (322 条)
    ↓
第三层：证据句提取
    ↓ 保留判定依据
最终数据集
```

**过滤效果**：

| 处理阶段 | 数据量 | 说明 |
|----------|--------|------|
| 原始关联标记 | 3,044 对 | 存在大量误判 |
| 同句共现过滤后 | 322 对 | 精确度大幅提升 |
| 最终数据集 | 180 家公司间的关系 | 每条均有证据支撑 |

## 三、关系类型识别

### 设计的关系分类体系

根据财经新闻的特点，设计了以下关系类型：

| 关系类型 | 判断依据 | 典型场景 |
|----------|----------|----------|
| 起诉 | "原告""被告""诉讼"等 | 合同纠纷、知识产权诉讼 |
| 保荐/承销 | "保荐机构""承销商"等 | IPO、再融资项目 |
| 投资/收购 | "收购""入股""参股"等 | 并购重组、战略投资 |
| 子公司/控股 | "子公司""控股"等 | 企业架构关系 |
| 合作 | "战略合作""签署协议"等 | 业务合作项目 |
| 供应/采购 | "供应商""客户"等 | 供应链关系 |
| 减持/增持 | "减持""增持"等 | 股权变动 |
| 竞争 | "竞争对手""同业竞争"等 | 市场竞争关系 |

### 关系分布统计

| 关系类型 | 数量 | 占比 |
|----------|------|------|
| 同事件提及 | 214 条 | 33% |
| 保荐/承销 | 30 条 | 9% |
| 投资/收购 | 28 条 | 9% |
| 子公司/控股 | 22 条 | 7% |
| 供应/采购 | 14 条 | 4% |
| 起诉 | 6 条 | 2% |
| 其他 | 8 条 | 2% |

## 四、功能模块设计

### 整体功能架构

```
┌─────────────────────────────────────────────────────┐
│                  用户交互层                          │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  │
│  │  实体选择   │  │  关系展示   │  │  新闻详情   │  │
│  │  模块       │  │  模块       │  │  展示模块   │  │
│  └─────────────┘  └─────────────┘  └─────────────┘  │
└────────────────────────┬────────────────────────────┘
                         │
┌────────────────────────▼────────────────────────────┐
│                  业务逻辑层                          │
│  • 公司搜索与匹配                                    │
│  • 关系判定与查询                                    │
│  • 数据动态加载                                      │
└────────────────────────┬────────────────────────────┘
                         │
┌────────────────────────▼────────────────────────────┐
│                  数据服务层                          │
│  ┌──────────────────────────────────────────────┐  │
│  │  离线数据集 (820家公司 / 322条关系)          │  │
│  └──────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

### 核心功能点

| 功能模块 | 实现要点 | 用户体验优化 |
|----------|----------|--------------|
| 公司搜索 | 支持模糊匹配，输入关键词即可搜索 | 无需输入完整公司名 |
| 智能推荐 | 公司2下拉框优先显示有关系公司 | 并标注关系类型 |
| 滚动加载 | 下拉选项分批加载，每次 10 条 | 避免 800+ 选项卡顿 |
| 结果展示 | 三元组 + 证据句 + 新闻详情 | 直观呈现判定依据 |
| 无关系提示 | 检测到无关系时友好提示 | 避免用户困惑 |

## 五、界面展示

### 5.1 实体选择界面

**公司1选择**：

- 点击下拉框，自动展示有关系的 100 家公司
- 输入关键词，实时模糊匹配所有 820 家公司

**公司2选择**：

- 与公司1有关系的公司排在最前面，并显示关系类型
- 例如：`科美诊断技术股份有限公司 [起诉]`
- 后续显示其他公司，方便探索潜在关系

### 5.2 关系抽取结果

展示内容：

- **三元组**：公司A → 关系类型 → 公司B
- **证据句**：判定该关系的原文句子
- **新闻详情**：标题、来源、时间、摘要、正文、原文链接

### 5.3 无关系处理

当两家公司无关系时：

- 显示友好提示："根据现有数据，这两家公司之间未找到明确的关系"
- 不展示新闻内容，避免误导用户

## 七、成果总结

### 定量成果

- 处理新闻数据 **20,000 条**
- 识别公司 **820 家**
- 抽取关系 **322 条**（含 10 种关系类型）
- 每条关系均附有**证据句**和**新闻来源**

### 定性成果

- 实现了**端到端**的跨文档实体关系抽取流程
- 建立了**可复用**的数据处理管道
- 提供了**用户友好**的交互界面

### 创新点

- 设计了针对财经新闻的关系过滤规则
- 实现了关系类型的自动标注
- 支持证据溯源，提升结果可信度
